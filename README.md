# VBA 開発プロジェクト構成

このリポジトリは、Excel/Access などの VBA 開発を前提としたディレクトリ構成を提供します。

## ディレクトリ概要

- `src/` : VBA ソースの置き場
  - `modules/` : 標準モジュール（.bas）
  - `class_modules/` : クラスモジュール（.cls）
  - `forms/` : ユーザーフォーム（.frm / .frx）
  - `references/` : 参照情報や依存関係メモ（.txt など）
- `tests/` : テスト用の Excel/Access ファイルや簡易テストスクリプト
- `data/` : 検証用データ、サンプル入力
- `samples/` : 実行例やデモファイル
- `build/` : 生成物やパッケージ出力先
- `export/` : VBA エクスポートファイルの一時配置
- `scripts/` : エクスポート/インポート自動化スクリプト
- `docs/` : 仕様書、設計書、手順書

## 推奨ワークフロー

1. VBA プロジェクトからモジュールを `export/` にエクスポートする。
2. `export/` の内容を整理して `src/` 配下に移動する。
3. `scripts/` に配置したスクリプトで再インポートを自動化する。
4. 仕様や設計は `docs/` にまとめる。

必要に応じて、プロジェクト固有のファイルやサブディレクトリを追加してください。

## VBA ソースコードの配置とファイル形式

PDF から取り込む場合は、まず VBA ソースを下記の形式に分割して保存することを推奨します。

- 標準モジュール: `src/modules/*.bas`
- クラスモジュール: `src/class_modules/*.cls`
- ユーザーフォーム: `src/forms/*.frm` と対応する `.frx`
- 参照情報・メモ: `src/references/*.txt`

PDF 原本は記録用として `docs/` に置き、必要に応じて `docs/source_pdf/` のようにサブフォルダを作成してください。
